Git

  • Git is a versioning tool used to keep track of changes in repositories
  • Git allows multiple team members to work on the same codebase without conflicting with eachother
  • Code can be cloned from a remote repository down to a local computer to get a copy of the code
  • Changes can then be made and added to a code branch, which can then be pushed back up
  • Code branches can be reviewed and merged back into the main code branch
  • This workflow allows multiple team members to work on the same codebase at the same time without causing conflicts or overwriting eachothers work
